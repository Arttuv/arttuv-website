Reference types in Java

2017-06-15
Writings
Different reference types in Java provide access to the object lifecycle, giving us tools for object reuse (object pools) and even a possibility to achieve a reliable object finalization. Differences between references can be explained by reachability. This is how Java API Specification defines reachability: Reachability from strongest to weakest: An object is strongly reachable if it can be reached by some thread without traversing any reference objects.
